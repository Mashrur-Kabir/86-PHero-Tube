JavaScript Concepts

Scope of var, let and const:

var: The var keyword declares a variable with function scope or globally scoped if not within a function. Variables declared with var are hoisted, meaning they are available before the actual declaration, though they are initialized as undefined until the declaration line is executed.

let: The let keyword declares a block-scoped variable, which means the variable is only accessible within the block it is defined in. let is not hoisted in the same way as var, so it cannot be accessed before its declaration.

const: The const keyword also declares a block-scoped variable, but unlike let, the value assigned to a const variable cannot be reassigned. However, if the const variable is an object or array, the contents of that object or array can be modified.

Use Cases of null and undefined:

null: null is an assignment value that represents the intentional absence of any object value. It is typically used to indicate that a variable should have an empty or non-existent value.

undefined: undefined is a value that JavaScript assigns to a variable that has been declared but not yet assigned a value. It is also the default return value of functions that do not explicitly return anything.

What is a REST API?:

A REST API (Representational State Transfer Application Programming Interface) is a set of rules and conventions for building and interacting with web services. REST APIs use HTTP methods such as GET, POST, PUT, DELETE, etc., to perform operations on resources identified by URLs. RESTful services are stateless, meaning that each request from a client contains all the information needed to process the request, without relying on any stored context on the server.
